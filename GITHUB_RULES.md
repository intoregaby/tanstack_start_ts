## Rules to Follow on EVERY Project

### RULE 1 — YOLO Badge
When merging any fix directly on my own repo, always 
merge PRs without requesting a review.
→ This earns the YOLO badge.

### RULE 2 — Pull Shark Badge  
Every feature or fix must be done in a separate branch 
and submitted as a Pull Request, never push directly 
to main. Always create a PR even for small changes.
→ This earns Pull Shark (2 merged PRs = bronze, 
  16 = silver, 128 = gold).

### RULE 3 — Pair Extraordinaire Badge
Every commit message must end with this co-author line:

Co-authored-by: intoregaby <intoremusinga@gmail.com>

→ This earns the Pair Extraordinaire badge.

### RULE 4 — Quickdraw Badge
After creating any issue or PR, if it is no longer 
needed or is immediately resolved, close it within 
5 minutes of opening.
→ This earns the Quickdraw badge.

### RULE 5 — Galaxy Brain Badge
For every major feature added, create a GitHub 
Discussion in the repo explaining what it does 
and why. Then post the answer and mark it as 
the correct answer.
→ This earns the Galaxy Brain badge.

### RULE 6 — Starstruck Badge
Every README.md must be professional, clear, and 
include: project description, tech stack, setup 
instructions, screenshots, and live demo link.
A good README attracts stars.
→ This earns Starstruck (16 stars = bronze).

### RULE 7 — Commit Consistency
Make at least 1 meaningful commit per working session.
Commit messages must follow this format:
  type: short description

  Co-authored-by: intoregaby <intoremusinga@gmail.com>

Types: feat / fix / docs / style / refactor / chore

### RULE 8 — Branch Naming
Always create branches in this format:
  feat/feature-name
  fix/bug-name
  docs/update-name
